Global Feed. Component Spec
FilterChip Atomic
TypePill toggle button, non-radio (one active at a time)
OptionsAll · Near Me · This Week
ActiveGold tint bg, gold border — same accent as stars
InactiveTransparent bg, muted border, tertiary text
FontDM Mono 500 · 7px · 0.1em tracking · uppercase
P1 noteNear Me requires location permission — prompt on first tap
WorldBar Atomic
PurposeAmbient social proof — show the network is alive and global
DataLogs this week (count) · Countries represented (count)
EmphasisNumbers in accent-gold, labels in text-tertiary
UpdateLive via WebSocket when prototype; static in mockup
GeoTag Atomic
PlacementInline in user-row, right of timestamp
ContentCity name only (not full address). User-set on log.
Dark#555 text · #1E1E1E border · no fill
Light#9A8C74 text · #C8BCA0 border · no fill
PrivacyCity-level only. Never coordinates. User can omit.
RareBadge Atomic
TriggerSong not played in last 20 setlists for that artist (Setlist.fm threshold)
PlacementInline in card footer, after song count stat
StyleGold tint bg · gold border · gold text — premium signal
PriorityP1 — requires Setlist.fm API integration for auto-detection
GlobalFeedCard Molecule
Differencevs. FriendsFeedCard: no photo panel (community photos are P1)
AvatarAlways avatar-init (initials) for strangers — no profile photo shown
EngagementHigher ♥ / ↩ counts — shows global reach, more discovery incentive
Follow CTAP1 — "Follow" ghost button on card, not in MVP
GlobalFeedScreen Organism
Default tabGlobal active (vs. Friends Feed which defaults to Friends)
Empty stateFriends tab shows "Follow people to see their logs" when 0 friends
SortReverse-chron default. "Near Me" filters by user location. "This Week" expands window.
Scroll580px max-height scroll container, hidden scrollbar
TransitionsTab switch = instant (no animation). Prototype shell will add slide.
