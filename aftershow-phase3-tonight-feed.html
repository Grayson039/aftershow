Tonight Feed. Component Spec
LiveHeaderBar Atomic
ContentPulsing red dot · "X shows happening now" · "in Y cities"
Dotblink animation 1.4s ease-in-out · #C4401C · 7px diameter
Count textDM Mono 500 · 8px · #C4401C both themes
Cities textDM Mono 400 · 7px · text-tertiary · right-aligned
DataLive count via WebSocket in prototype. Static in mockup.
ShowProgressBar Atomic
PurposeShows how far into the set the logger is. Scannable at a glance.
TrackFull width · 2px height · bg-border color
Fill#C4401C · width = (songs logged / expected total) × 100%
LabelDM Mono · 7px · "[N] songs · [context]" — context changes as show progresses: "mid-set", "encore soon", "~Xh Xm in"
ExpectedDerived from Setlist.fm average set length for artist. P1 — static in MVP.
LiveSetlistPreview Molecule
ContentLast 3 songs logged · song number + title · "updating" indicator
Latest songBottom song highlighted in accent-gold — most recent entry
Update dot4px red dot · blink animation 2s · sits beside "updating" label
Footer"+ N more expected ›" or "enc. expected ›" based on progress
BackgroundSlightly darker than card bg — inset panel feel
FontSong titles: Crimson Pro Italic 400 · 13px · secondary text color
Tap actionOpens full Show Page (P1 — not in MVP feed)
LiveCard Molecule
vs. FeedCardAdds: ShowProgressBar, LiveSetlistPreview, red border glow on active-show state
BorderActive show: rgba(196,64,28,.45) border + subtle red box-shadow
Footer stat"● live" replaces song count while show is in progress
No ratingStars not shown until show wraps — rating filled at end of log
WrappedCard Molecule
PurposeShows that ended tonight but aren't full archive posts yet
BadgeDONE — muted border, tertiary text. Contrast with LIVE badge.
Opacity0.7 dark / 0.75 light — visually recedes below LIVE cards
RatingStars shown — rating was submitted when show ended
TransitionLiveCard → WrappedCard when logger taps "End Show" in live logging mode
TonightFeedScreen Organism
Default tabTonight active · Friends shows empty state ("no one out tonight") · Global shows abbreviated list
SortLIVE cards first (newest → oldest by start time) · Just Wrapped below divider
Load more"42 more shows tonight ↓" — pagination, not infinite scroll
RefreshPull-to-refresh (P1). Live cards auto-update setlist via WebSocket in prototype.
